OSSERVAZIONI SUGLI ALGORITMI DI ORDINAMENTO

Dal grafico iniziale, "grafico_tempi_iniziale", si vede che esiste una soglia sotto alla quale un algoritmo quadratico è migliore di Quick Sort, ovvero Insertion Sort. Infatti:
- per n≲10–15: gli algoritmi quadratici (O(n²)) Bubble Sort, Selection Sort e soprattutto Insertion Sort sembrano più veloci;
- per n≳20–30: gli algoritmi Merge Sort e Quick Sort (O(nlogn)) diventano stabilmente migliori.
Dato che dall'immagine ho notato che l’algoritmo quadratico migliore è Insertion Sort, lo userò sotto la soglia dove è più veloce di Quick Sort per migliorarlo.
La versione modificata di Quick Sort l'ho implementata nel "sort_new.hpp" per fare poi il grafico finale, ma la riporto anche qui sotto:
/* insertion_sort per il quick_sort per n sotto la soglia: uguale a prima ma deve avere 3 parametri in entrata */
template<typename T>
void insertion_sort(std::vector<T>& A, int p, int r)
{
    for (int j=p+1; j<=r; j+=1) {
        T key=A[j];
        int i=j-1;
        while (i>=p && A[i]>key) {
            A[i+1]=A[i];
            i=i-1;
        }
        A[i+1]=key;
    }
}
const int soglia=16; /* soglia per migliorare Quick Sort sotto la quale è più lento di Insertion Sort */
template<typename T>
void quick_sort(std::vector<T>& A, int p, int r)
{
    while (p<r) {
        /* per n piccolo uso insertion sort */
        if (r-p+1<=soglia) {
            insertion_sort(A,p,r);
            return;
        }
        int q=partition(A,p,r);
        quick_sort(A,p,q-1);
		quick_sort(A,q+1,r);
    }
}

Nel nuovo grafico, "grafico_tempi_finale", confronto i tempi tra Bubble Sort, Selection Sort, Insertion Sort, Merge Sort, la nuova versione di Quick Sort e std::sort.
Posso osservare che la nuova implementazione di Quick Sort permette di ridurre il tempo per vettori piccoli, diventando così tra i più veloci ed andando a sovrapporsi ad Insertion Sort (dove invece in quello iniziale era sopra, quindi più lento).
Osservo sempre anche che Bubble Sort, Selection Sort, Insertion Sort crescono velocemente (O(n²)), invece Merge Sort, Quick Sort crescono lentamente (O(nlogn)):
infatti per n sotto la soglia gli algoritmi quadratici risultano competitivi, per n grande sono migliori Merge Sort e Quick Sort.
Infine, std::sort risulta generalmente il più efficiente.
